# Sweatfile Tommy Codegen Migration

**Date:** 2026-03-22 **Issue:** #44 **Status:** approved

## Problem

`packages/spinclass/internal/sweatfile/` uses BurntSushi/toml for TOML parsing
and encoding. `Save()` drops all user comments because `toml.NewEncoder`
serializes from struct fields only. Future spinclass features will mutate
sweatfiles programmatically, so comment-preserving round-trips are required.

## Solution

Replace BurntSushi/toml with tommy codegen (`//go:generate tommy generate`).
Tommy generates a `SweatfileDocument` type backed by a CST that preserves
comments and formatting across decode-modify-encode cycles.

Thread `*SweatfileDocument` as the primary type through the parse/load/save API,
exposing `Data() *Sweatfile` for callers that only need the struct.

## API Changes

Before (method on Sweatfile):

``` go
func (sf *Sweatfile) Parse(data []byte) error
func (sf *Sweatfile) Load(path string) error
func (sf Sweatfile) Save(path string) error
```

After (free functions returning document, Save as document method):

``` go
func Parse(data []byte) (*SweatfileDocument, error)
func Load(path string) (*SweatfileDocument, error)
func (*SweatfileDocument) Save(path string) error
func (*SweatfileDocument) Data() *Sweatfile          // generated by tommy
func (*SweatfileDocument) Undecoded() []string        // generated by tommy
func (*SweatfileDocument) Encode() ([]byte, error)    // generated by tommy
```

## Files Unchanged

- `hierarchy.go` merge logic (operates on `Sweatfile` structs)
- `apply.go` (receives merged `Sweatfile` struct)
- `util.go`

## Caller Updates

**hierarchy.go** (2 sites): `sf.Load(path)` becomes `doc, err := Load(path)`,
then `doc.Data()` feeds into merge. `LoadWorktreeHierarchy` same pattern.

**validate.go** (2 sites): `parsed.Parse(data)` becomes
`doc, err := Parse(data)`, use `doc.Data()` for field checks, `doc.Undecoded()`
replaces `toml.Decode` + `md.Undecoded()`.

**sweatfile_test.go** (\~19 sites): `var sf Sweatfile; sf.Parse(data)` becomes
`doc, err := Parse(data); sf := doc.Data()`. Mechanical.

## Rollback

Single commit. `git revert <sha>` restores BurntSushi/toml, removes generated
file. No dual-architecture needed --- this is an internal implementation swap
behind the same package boundary.

## Lessons from Moxy Migration

1.  Round-trip test: decode-encode with no modifications must produce
    byte-identical output
2.  Generated file is committed per Go convention
3.  Run `just go-mod-sync` after dependency changes
4.  Test both flat-key and sub-table forms for `*Hooks` pointer-to-struct field
